## pattern 4: message array construction [EXTEND]

### location

`src/domain.operations/atom/genBrainAtom.ts`

### description

builds messages array with system prompt and prior exchanges.

### citation [8]: message construction

```ts
const messages: OpenAI.ChatCompletionMessageParam[] = [];
if (systemPrompt) {
  messages.push({ role: 'system', content: systemPrompt });
}
if (askInput.on?.episode) {
  for (const exchange of askInput.on.episode.exchanges) {
    messages.push({ role: 'user', content: exchange.input });
    messages.push({ role: 'assistant', content: exchange.output });
  }
}
messages.push({ role: 'user', content: askInput.prompt });
```

- file: `src/domain.operations/atom/genBrainAtom.ts:80-90`
- action: **[EXTEND]** — when prompt is BrainPlugToolExecution[], add tool messages via cast function

## new patterns needed from xai reference

### pattern 7: cast functions [NEW]

based on xai pr #15, need three cast functions:

**castIntoTogetherToolDef**
- converts BrainPlugToolDefinition to openai-compatible tool format
- transforms zod schema to JSON schema

**castFromTogetherToolCall**
- converts openai tool_calls response to BrainPlugToolInvocation
- maps id → exid, parses JSON arguments

**castIntoTogetherToolMessages**
- converts BrainPlugToolExecution[] to openai tool message format
- maps signal to role: "tool" messages

### pattern 8: TPlugs generic [NEW]

from xai implementation:
- factory returns `BrainAtom<TPlugs>`
- ask function typed with TPlugs for type-safe tool invocations
- output narrowed based on tools presence
