# blackbox criteria: brain atom tool use

## usecase.1 = tool invocation

given a brain atom with tools plugged
  when ask is called with a prompt that requires tool use
    then result.calls.tools contains tool invocations
      sothat caller can execute the requested tools
    then result.output is null
      sothat caller knows brain deferred to tools
    then each invocation has exid, slug, input
      sothat caller can correlate results back to requests
    then invocation.input is typed per tool schema
      sothat caller can pass input directly to tool executor

given a brain atom with tools plugged
  when ask is called with a prompt brain can answer directly
    then result.output contains the answer
      sothat caller receives direct response
    then result.calls is null
      sothat caller knows no tools were requested

## usecase.2 = tool continuation

given a brain atom that requested tools in prior exchange
  when ask is called with BrainPlugToolExecution[] as prompt
    then result processes tool results
      sothat brain can synthesize final answer
    then episode.exchanges accumulates the tool exchange
      sothat conversation history is preserved

given a brain atom that requested tools
  when tool execution returns signal: 'success'
    then brain receives the tool output
      sothat brain can use the result in synthesis

given a brain atom that requested tools
  when tool execution returns signal: 'error:constraint'
    then brain receives the error context
      sothat brain can adjust approach or report failure

given a brain atom that requested tools
  when tool execution returns signal: 'error:malfunction'
    then brain receives the error context
      sothat brain can handle system failure gracefully

## usecase.3 = structured output with tools

given a brain atom with tools plugged AND output schema
  when ask is called
    then response_format json_schema is sent to provider
      sothat output is provider-validated JSON
    then result.output (when not null) matches schema
      sothat caller receives typed output

given a brain atom with tools plugged BUT no response_format
  when ask is called
    then brain returns free-form text response
      sothat models without json_schema support work
    then result.output is parsed client-side
      sothat rhachet contract is satisfied

## usecase.4 = type safety

given tool definitions with zod schemas
  when tools are passed to plugs.tools
    then tool schemas are converted to JSON schema
      sothat provider receives valid tool definitions

given result.calls.tools with invocations
  when caller accesses invocation.input
    then input is typed per the tool's input schema
      sothat type safety flows through the tool loop

## usecase.5 = metrics and observability

given a tool invocation exchange
  when ask completes (with or without tools)
    then metrics.cost.time tracks elapsed duration
      sothat caller can observe performance
    then metrics.cost.cash tracks token costs
      sothat caller can observe spend
    then metrics.size.tokens tracks input/output tokens
      sothat caller can monitor usage
