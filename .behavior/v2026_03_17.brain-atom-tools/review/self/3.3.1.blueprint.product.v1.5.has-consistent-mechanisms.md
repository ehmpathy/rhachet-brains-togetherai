| function | xai name | together name | pattern match? |

**verified patterns**:

1. **names follow convention**: `cast{Into|From}{Provider}{Type}`
   - `castIntoTogetherToolDef` — into provider format
   - `castFromTogetherToolCall` — from provider format
   - `castIntoTogetherToolMessages` — into provider format

   - `Into` = rhachet → provider
   - `From` = provider → rhachet

3. **type-safe boundaries**: each uses both rhachet types and OpenAI types
   - input: rhachet type (BrainPlugToolDefinition, etc)
   - output: OpenAI type (ChatCompletionTool, etc)

4. **placement in infra/cast/**: follows architectural pattern from xai

**verdict**: cast functions are consistent with established xai patterns. no duplicate mechanisms.

## duplication check

### grep search: where is z.toJSONSchema used?

```bash
grep -r "toJSONSchema" src/
```

**results**:
- `src/infra/cast/castIntoTogetherToolDef.ts:17` — tool input schema
- `src/domain.operations/atom/genBrainAtom.ts:130` — output schema

**analysis**:
these are TWO DIFFERENT purposes:
1. `castIntoTogetherToolDef` converts **tool.schema.input** to JSON schema for function parameters
2. `genBrainAtom` converts **askInput.schema.output** to JSON schema for response_format

could we consolidate? no — they operate on different schemas at different points:
- tool def conversion happens BEFORE the api call (line 134)
- output schema conversion happens DURING api call setup (line 130)

**verdict**: ✅ no duplication — different schemas, different purposes.

## lessons from this review

1. **reference implementation guides consistency**: the xai PR #15 established patterns that this implementation follows exactly.

2. **infra/cast/ is the right location**: cast functions are infrastructure, not domain operations. the placement is consistent.

3. **reuse over duplication**: genBrainAtom reuses extant codepaths (openai client, metrics, episode) rather than re-implement.

4. **single responsibility per cast**: each cast handles one direction of transformation. no multi-purpose casts.
