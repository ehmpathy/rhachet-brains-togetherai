- cast functions live in infra/cast/ — correct layer for adapters between external API shapes
- genBrainAtom stays in domain.operations/atom/ — correct layer for domain behavior
- imports flow downward: domain.operations → infra → external types

### procedure standards hold because:
- all cast functions use `(input: { ... })` pattern with inline types
- all functions are arrow functions, no function keyword
- each cast function has single responsibility: one shape transform

### type standards hold because:
- z.toJSONSchema() produces valid JSON schema from zod
- no `as` casts needed — types flow naturally from rhachet to OpenAI shapes
- BrainPlugToolInvocation maps cleanly from ChatCompletionMessageToolCall

### test standards hold because:
- test cases follow [caseN][tN] format: case5, case6, case7, etc.
- given-when-then structure from test-fns used

- genBrainAtom uses gen verb (not create, make, build)
- variable names use noun-adj order where applicable
- no gerunds in function or variable names
