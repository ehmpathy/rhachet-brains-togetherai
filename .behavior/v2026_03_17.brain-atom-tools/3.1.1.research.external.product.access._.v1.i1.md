## together ai function call api

### contract

together ai provides an openai-compatible function call api. the key parameters are:

- `tools`: array of function definitions with name, description, and JSON schema parameters
- `tool_choice`: controls tool selection behavior (`auto`, `none`, `required`, or specific function)
- `response_format`: optional json_schema for structured output alongside tools

### citation [1]: together ai function call documentation

> "to use function calling, you need to define tools in the api request. each tool is a function with a name, description, and parameters schema."

- source: [together ai docs - function call](https://docs.together.ai/docs/function-calling)
- date: 2025
- credibility: [official]

### citation [2]: together ai tool_choice parameter

> "you can set tool_choice to force a specific function using `tool_choice= {"type": "function", "function": {"name": "<function_name>"}}`, or prevent function calls entirely using `tool_choice="none"`."

- source: [together ai docs - function call](https://docs.together.ai/docs/function-calling)
- date: 2025
- credibility: [official]

### citation [3]: together ai function call announcement

> "today, we are excited to announce function calling and json mode for open source models on together ai... function calling enables structured interactions with llms by combining function call and schema-constrained json outputs."

- source: [together ai blog - function call json mode](https://www.together.ai/blog/function-calling-json-mode)
- date: 2024-06
- credibility: [official]

## JSON schema conversion from zod

### citation [8]: zod native JSON schema support

> "zod 4 introduces first-party JSON schema conversion via z.toJSONSchema(). a second argument can be used to customize the conversion logic. you can target different JSON schema versions like draft-2020-12, draft-07, draft-04, and openapi-3.0."

- source: [zod documentation - JSON schema](https://zod.dev/json-schema)
- date: 2025
- credibility: [official]

### citation [9]: zod-to-json-schema npm package

> "converts zod schemas into JSON schemas, supports all relevant schema types, basic string, number and array length validations and string patterns, and resolves recursive and recurring schemas with internal $refs. as of november 2025, this project will no longer be receiving updates. zod v4 natively supports generating JSON schemas."

- source: [npm - zod-to-json-schema](https://www.npmjs.com/package/zod-to-json-schema)
- date: 2025-11
- credibility: [official]

**convergence signal**: sources [8] and [9] confirm zod 4+ has native JSON schema conversion. for zod 3, the zod-to-json-schema package provides the same capability.

## ReAct pattern for tool loops

### citation [13]: ReAct framework definition

> "ReAct (Reasoning + Acting) is a framework introduced by Yao et al., 2022 where llms are used to generate both reasoning traces and task-specific actions in an interleaved manner."

- source: [prompt engineering guide - ReAct](https://www.promptingguide.ai/techniques/react)
- date: 2024
- credibility: [tutorial]

### citation [14]: ReAct loop structure

> "a ReAct agent operates in a loop of thought, action, observation, repeating until it reaches a solution. the agent analyzes the user's query, decides on a tool to call, executes it, and reevaluates progress."

- source: [ibm - what is a ReAct agent](https://www.ibm.com/think/topics/react-agent)
- date: 2025
- credibility: [official]

### citation [15]: ReAct implementation

> "implementation requires a language model to serve as the agent's brain, some tools the agent can use, and a loop that ties them together."

- source: [simon willison - python ReAct pattern](https://til.simonwillison.net/llms/python-react-pattern)
- date: 2024
- credibility: [practitioner]

**convergence signal**: sources [13], [14], [15] agree on the thought-action-observation loop pattern. the brain atom tool use implements this: ask with prompt, get tool calls, execute tools, continue with results.

## conflict detection

### response_format + tools compatibility

no conflict detected. sources [3] and [10] confirm that json_schema response_format can be used alongside tools. together ai's announcement explicitly mentions "combine function call and schema-constrained json outputs."

### zod version for JSON schema conversion

**note**: source [8] says zod 4 has native support, while source [9] says the zod-to-json-schema package is deprecated for zod 4. no conflict - just version-specific guidance. use native z.toJSONSchema() for zod 4+, use zod-to-json-schema for zod 3.

## summary of best practices

1. **define clear tool schemas**: use descriptive names, detailed descriptions, and precise JSON schema parameters [6]
2. **handle multiple tool calls**: assume responses may contain zero, one, or multiple calls [7]
3. **validate all inputs**: never execute tool inputs without validation [27]
4. **implement retry with backoff**: use exponential backoff with jitter for transient failures [22]
5. **apply privilege separation**: grant minimum required tools, scope permissions [26]
6. **use schema validation**: two-layer validation with schema check and retry [23]

## what NOT to do (anti-patterns)

1. **do not execute unvalidated tool inputs**: risk of prompt injection attacks [24], [25]
2. **do not assume single tool call**: always handle array of tool calls [7]
3. **do not grant excessive tool access**: minimum privilege principle [26]
4. **do not retry without backoff**: thunder herd problem [22]
5. **do not skip schema validation**: llms are probabilistic, may generate invalid calls [23]
