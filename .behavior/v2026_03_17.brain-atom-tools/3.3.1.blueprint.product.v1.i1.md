# blueprint: brain atom tool use

## summary

implement tool use capability for together ai brain atoms, match xai pr #15 patterns and test coverage.

delivers:
1. **cast functions** ‚Äî translate between rhachet tool types and openai-compatible format
2. **genBrainAtom extension** ‚Äî add TPlugs generic, conditional prompt type, tool calls in output
3. **integration tests** ‚Äî tool invocation, tool continuation, error signals, model compatibility

## type definitions

### TPlugs generic pattern

```ts
// ask function with TPlugs
ask: async <TOutput, TPlugs extends BrainPlugs = BrainPlugs>(
  askInput: {
    on?: { episode: BrainEpisode };
    plugs?: TPlugs;
    role: { briefs?: Artifact<typeof GitFile>[] };
    prompt: TPlugs extends { tools: unknown[] }
      ? string | BrainPlugToolExecution[]
      : string;
    schema: { output: z.Schema<TOutput> };
  },
  context?: Empty,
): Promise<BrainOutput<TOutput, 'atom', TPlugs>> => { ... }
```

### output type narrowed

```ts
// BrainOutput with tools
{
  output: TOutput | null;  // null when brain calls tools
  calls: {
    tools: BrainPlugToolInvocation[];
  } | null;  // null when brain produces output
  metrics: BrainOutputMetrics;
  episode: BrainEpisode;
  series: null;  // always null for atom
}
```

## contracts reference

### rhachet types used

from `rhachet/brains`:
- `BrainPlugs` ‚Äî plugs container type
- `BrainPlugToolDefinition` ‚Äî tool definition shape
- `BrainPlugToolInvocation` ‚Äî tool call from model
- `BrainPlugToolExecution` ‚Äî tool result to feed back
- `genBrainPlugToolDeclaration` ‚Äî factory for typed tool definitions

### openai types used

from `openai`:
- `OpenAI.ChatCompletionTool` ‚Äî tool definition for api
- `OpenAI.ChatCompletionMessageToolCall` ‚Äî tool call in response
- `OpenAI.ChatCompletionToolMessageParam` ‚Äî tool result message

## implementation phases

1. **phase 0: cast functions**
   - castIntoTogetherToolDef
   - castFromTogetherToolCall
   - castIntoTogetherToolMessages

2. **phase 1: genBrainAtom tool invocation**
   - add TPlugs generic
   - convert and send tools in api call
   - detect and return tool calls

3. **phase 2: genBrainAtom tool continuation**
   - accept BrainPlugToolExecution[] as prompt
   - convert to tool messages
   - accumulate in episode

4. **phase 3: integration tests**
   - tool invocation tests
   - tool continuation tests
   - error signal tests
   - model compatibility tests
